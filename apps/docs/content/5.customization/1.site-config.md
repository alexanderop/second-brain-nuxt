# Site Configuration

The `site.config.ts` file is the central configuration hub for customizing your Second Brain instance. All user-customizable values are consolidated here for easy modification.

## Configuration File

The configuration file is located at:

```
apps/web/site.config.ts
```

## Site Identity

Configure your site's branding and metadata:

```ts
export const siteConfig = {
  // Display name in header, page titles, and PWA
  name: 'Second Brain',

  // Short name for PWA app launcher (no spaces)
  shortName: 'SecondBrain',

  // SEO description and PWA manifest
  description: 'Personal knowledge base with connected notes',

  // Browser chrome and PWA theme color (hex format)
  themeColor: '#1a1a2e',

  // Allow search engine indexing (default: false)
  allowIndexing: false,
}
```

| Option | Type | Description |
|--------|------|-------------|
| `name` | `string` | Displayed in header, browser tab titles, and PWA manifest |
| `shortName` | `string` | Used in PWA app launchers, should have no spaces |
| `description` | `string` | Used for SEO meta tags and PWA manifest |
| `themeColor` | `string` | Hex color for browser chrome and mobile status bar |
| `allowIndexing` | `boolean` | When `false`, adds `noindex, nofollow` meta tags |

## Navigation

Customize the header navigation links:

```ts
nav: [
  { label: 'Home', to: '/', icon: 'i-lucide-home' },
  { label: 'Table', to: '/table', icon: 'i-lucide-table-2' },
  { label: 'Books', to: '/books', icon: 'i-lucide-book-open' },
  { label: 'Podcasts', to: '/podcasts', icon: 'i-lucide-podcast' },
  { label: 'Newsletters', to: '/newsletters', icon: 'i-lucide-newspaper' },
  { label: 'Graph', to: '/graph', icon: 'i-lucide-network' },
  { label: 'Stats', to: '/stats', icon: 'i-lucide-bar-chart-2' },
  { label: 'Tags', to: '/tags', icon: 'i-lucide-tags' },
  { label: 'Authors', to: '/authors', icon: 'i-lucide-users' },
  { label: 'About', to: '/about', icon: 'i-lucide-user' },
],
```

### Adding a Navigation Item

Add a new object to the `nav` array:

```ts
nav: [
  // ... existing items
  { label: 'Projects', to: '/projects', icon: 'i-lucide-folder-kanban' },
],
```

Each navigation item requires:

| Property | Type | Description |
|----------|------|-------------|
| `label` | `string` | Text displayed in the navigation |
| `to` | `string` | Route path (must match a page in `app/pages/`) |
| `icon` | `string` | Lucide icon class (format: `i-lucide-{icon-name}`) |

### Removing a Navigation Item

Simply delete the corresponding object from the `nav` array.

### Reordering Navigation

Change the order of objects in the array. Items appear left-to-right in desktop and top-to-bottom in mobile.

### Finding Icons

Icons use the [Lucide](https://lucide.dev/icons/) icon set. Browse available icons and use the format `i-lucide-{icon-name}` where `{icon-name}` uses kebab-case.

Examples:
- `i-lucide-home` - Home icon
- `i-lucide-book-open` - Book icon
- `i-lucide-folder-kanban` - Kanban folder icon
- `i-lucide-sparkles` - Sparkles icon

## Keyboard Shortcuts

Shortcuts are organized into three categories:

### Navigation Shortcuts

Two-key sequences for page navigation:

```ts
shortcuts: {
  navigation: [
    { keys: ['G', 'H'], to: '/', description: 'Go to home' },
    { keys: ['G', 'B'], to: '/books', description: 'Go to books' },
    { keys: ['G', 'P'], to: '/podcasts', description: 'Go to podcasts' },
    { keys: ['G', 'N'], to: '/newsletters', description: 'Go to newsletters' },
    { keys: ['G', 'G'], to: '/graph', description: 'Go to graph' },
    { keys: ['G', 'T'], to: '/tags', description: 'Go to tags' },
    { keys: ['G', 'A'], to: '/authors', description: 'Go to authors' },
    { keys: ['G', 'S'], to: '/stats', description: 'Go to stats' },
  ],
}
```

To use: Press `G` then the second key (e.g., `G` then `H` to go home).

### Action Shortcuts

Single-key actions for interacting with content:

```ts
actions: [
  { keys: ['J'], description: 'Next item' },
  { keys: ['K'], description: 'Previous item' },
  { keys: ['Enter'], description: 'Open selected' },
  { keys: ['O'], description: 'Open resource link' },
  { keys: ['A'], description: 'Open author page' },
  { keys: ['shift', 'L'], description: 'Copy wiki-link' },
  { keys: ['shift', 'C'], description: 'Copy URL' },
],
```

### General Shortcuts

Utility and mode shortcuts:

```ts
general: [
  { keys: ['?'], description: 'Show keyboard shortcuts' },
  { keys: ['meta', 'K'], description: 'Open search' },
  { keys: ['/'], description: 'Open search' },
  { keys: ['Esc'], description: 'Close modal / go back' },
  { keys: ['meta', 'alt', 'T'], description: 'Toggle dark mode' },
  { keys: ['meta', 'shift', '\\'], description: 'Toggle focus mode' },
  { keys: [']'], description: 'Toggle table of contents' },
],
```

### Adding Custom Shortcuts

Add a new entry to the appropriate category:

```ts
navigation: [
  // ... existing shortcuts
  { keys: ['G', 'R'], to: '/reading', description: 'Go to reading list' },
],
```

### Modifier Keys

Use these strings in the `keys` array:

| Key | Mac | Windows |
|-----|-----|---------|
| `meta` | `⌘ Command` | `Ctrl` |
| `alt` | `⌥ Option` | `Alt` |
| `shift` | `⇧ Shift` | `Shift` |
| `ctrl` | `^ Control` | `Ctrl` |

::alert{type="info"}
Navigation shortcuts automatically navigate to the specified `to` path. Action shortcuts require custom implementation in the component handling that context.
::

## Social Links

Configure external profile links:

```ts
social: {
  github: 'yourusername',
  twitter: 'yourusername',
},
```

These are used in the footer and about page. Leave empty strings to hide specific links.

### Adding More Social Links

You can extend the social object:

```ts
social: {
  github: 'yourusername',
  twitter: 'yourusername',
  linkedin: 'yourusername',
  youtube: '@yourchannel',
  website: 'https://yoursite.com',
},
```

::alert{type="warning"}
Adding new social fields requires updating components that use `useSiteConfig().social` to display them.
::

## Using Config in Components

Access configuration values using the `useSiteConfig()` composable:

```vue
<script setup lang="ts">
const config = useSiteConfig()
</script>

<template>
  <header>
    <h1>{{ config.name }}</h1>
    <nav>
      <NuxtLink
        v-for="item in config.nav"
        :key="item.to"
        :to="item.to"
      >
        <UIcon :name="item.icon" />
        {{ item.label }}
      </NuxtLink>
    </nav>
  </header>
</template>
```

### Page Titles

Use the `usePageTitle()` composable to set consistent page titles:

```vue
<script setup lang="ts">
// Sets title to "Books - Second Brain"
usePageTitle('Books')
</script>
```

## TypeScript Types

The configuration exports TypeScript types for type safety:

```ts
import type { SiteConfig, NavItem, ShortcutCategory } from '~~/site.config'

// Access nav item type
const item: NavItem = {
  label: 'Home',
  to: '/',
  icon: 'i-lucide-home'
}

// Shortcut categories: 'navigation' | 'actions' | 'general'
const category: ShortcutCategory = 'navigation'
```

## Complete Example

Here's a fully customized configuration:

```ts
export const siteConfig = {
  name: 'My Knowledge Garden',
  shortName: 'KnowledgeGarden',
  description: 'A digital garden for learning and growing',
  themeColor: '#2d5a27',
  allowIndexing: true,

  nav: [
    { label: 'Home', to: '/', icon: 'i-lucide-home' },
    { label: 'Library', to: '/books', icon: 'i-lucide-library' },
    { label: 'Notes', to: '/table', icon: 'i-lucide-notebook' },
    { label: 'Graph', to: '/graph', icon: 'i-lucide-network' },
  ],

  shortcuts: {
    navigation: [
      { keys: ['G', 'H'], to: '/', description: 'Go to home' },
      { keys: ['G', 'L'], to: '/books', description: 'Go to library' },
      { keys: ['G', 'N'], to: '/table', description: 'Go to notes' },
      { keys: ['G', 'G'], to: '/graph', description: 'Go to graph' },
    ],
    actions: [
      { keys: ['J'], description: 'Next item' },
      { keys: ['K'], description: 'Previous item' },
      { keys: ['Enter'], description: 'Open selected' },
    ],
    general: [
      { keys: ['?'], description: 'Show keyboard shortcuts' },
      { keys: ['meta', 'K'], description: 'Open search' },
      { keys: ['/'], description: 'Open search' },
    ],
  },

  social: {
    github: 'myusername',
    twitter: 'myhandle',
  },
} as const
```
