# Content Types

Every content item has a `type` field that determines its behavior, required fields, and UI presentation. Types are divided into **external** (content from others) and **personal** (your own notes).

## Common Fields

These fields are available for all content types:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | **Yes** | Display name |
| `type` | string | **Yes** | Content type identifier |
| `url` | string | No | Link to original source |
| `cover` | string | No | Cover image URL |
| `tags` | string[] | No | Categorization tags |
| `authors` | string[] | **External only** | Author slugs |
| `summary` | string | No | AI-generated or manual summary |
| `notes` | string | No | Personal thoughts |
| `date` | string | No | Date added |
| `rating` | number | No | Quality rating (1-10) |

---

## External Content Types

External types represent content created by others. **All external types require the `authors` field.**

### YouTube

Video content from YouTube.

```yaml
---
title: "Building a Second Brain - Full Workshop"
type: youtube
url: https://www.youtube.com/watch?v=example
authors: [tiago-forte]
tags: [productivity, note-taking]
summary: Complete workshop on the PARA method and building your second brain.
---
```

### Podcast

Podcast episode content. Links to a podcast show via the `podcast` field.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `podcast` | string | No | Podcast show slug |
| `guests` | string[] | No | Guest author slugs |
| `urls` | object[] | No | Platform-specific URLs |

```yaml
---
title: "#123 - Atomic Habits"
type: podcast
podcast: huberman-lab
authors: [andrew-huberman]
guests: [james-clear]
tags: [habits, neuroscience]
urls:
  - platform: spotify
    url: https://open.spotify.com/episode/...
  - platform: youtube
    url: https://www.youtube.com/watch?v=...
---
```

### Article

Written articles from blogs, news sites, or publications.

```yaml
---
title: "Why Greatness Cannot Be Planned"
type: article
url: https://example.com/article
authors: [kenneth-stanley]
tags: [ai, creativity]
summary: Exploration of novelty search and why objectives can be counterproductive.
---
```

### Book

Books with reading status tracking.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `readingStatus` | string | No | `want-to-read`, `reading`, or `finished` |
| `startedReading` | string | No | Date started (YYYY-MM-DD) |
| `finishedReading` | string | No | Date finished (YYYY-MM-DD) |

```yaml
---
title: Atomic Habits
type: book
authors: [james-clear]
tags: [productivity, habits, self-improvement]
cover: https://covers.example.com/atomic-habits.jpg
rating: 7
readingStatus: finished
startedReading: "2024-01-15"
finishedReading: "2024-02-01"
---
```

### Manga

Manga series with volume and publication status tracking.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `volumes` | number | **Yes** | Total volume count |
| `status` | string | **Yes** | `ongoing`, `completed`, or `hiatus` |

```yaml
---
title: One Piece
type: manga
authors: [eiichiro-oda]
tags: [adventure, shounen]
cover: https://covers.example.com/one-piece.jpg
volumes: 107
status: ongoing
rating: 9
---
```

### Movie

Film content with watch tracking.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `watchingStatus` | string | No | `want-to-watch`, `watching`, or `watched` |
| `watchedOn` | string | No | Date watched (YYYY-MM-DD) |
| `trailer` | string | No | Trailer URL |

```yaml
---
title: Oppenheimer
type: movie
authors: [christopher-nolan]
tags: [history, biography, drama]
url: https://www.imdb.com/title/tt15398776/
cover: https://posters.example.com/oppenheimer.jpg
watchingStatus: watched
watchedOn: "2024-07-21"
rating: 8
---
```

### TV

Television series content.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `watchingStatus` | string | No | `want-to-watch`, `watching`, or `watched` |
| `watchedOn` | string | No | Date finished watching |

```yaml
---
title: Breaking Bad
type: tv
authors: [vince-gilligan]
tags: [drama, crime]
watchingStatus: watched
rating: 10
---
```

### Tweet

Individual tweet content (also has its own collection).

```yaml
---
title: "Thread on compound interest"
type: tweet
authors: [naval]
tags: [wealth, investing]
url: https://twitter.com/naval/status/...
---
```

### Course

Online courses and educational content.

```yaml
---
title: "CS50: Introduction to Computer Science"
type: course
url: https://cs50.harvard.edu/
authors: [david-malan]
tags: [programming, computer-science]
summary: Harvard's intro to CS covering algorithms, data structures, and web development.

```

### Reddit

Reddit posts and discussions.

```yaml
---
title: "Ask HN: Best resources for learning system design?"
type: reddit
url: https://reddit.com/r/programming/comments/...
authors: [reddit-user]
tags: [system-design, learning]
---
```

### GitHub

GitHub repositories.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `stars` | number | No | Repository star count |
| `language` | string | No | Primary programming language |

```yaml
---
title: "Vue.js - The Progressive JavaScript Framework"
type: github
url: https://github.com/vuejs/vue
authors: [evan-you]
tags: [javascript, framework, frontend]
stars: 45000
language: TypeScript
---
```

### Newsletter

Newsletter article content. Links to a newsletter via the `newsletter` field.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `newsletter` | string | No | Newsletter slug |
| `issueNumber` | number | No | Issue number |
| `guest_author` | string | No | Guest author slug |

```yaml
---
title: "The Art of Doing Science and Engineering"
type: newsletter
newsletter: brain-food
authors: [shane-parrish]
issueNumber: 512
tags: [learning, science]
url: https://fs.blog/newsletter/512
---
```

## Personal Content Types

Personal types are for your own notes and don't require authors.

### Quote

Memorable quotes worth saving.

```yaml
---
title: "On Compound Knowledge"
type: quote
tags: [learning, wisdom]
authors: [naval] # Optional - attribute to source
---

"Read what you love until you love to read."
```

### Note

General notes and fleeting thoughts.

```yaml
---
title: Morning Routine Ideas
type: note
tags: [productivity, habits]
---

Ideas for improving my morning routine...
```

### Evergreen

Polished, permanent notes that represent distilled knowledge.

```yaml
---
title: Spaced Repetition Works Because of the Forgetting Curve
type: evergreen
tags: [learning, memory]
---

Evergreen notes are densely linked and refined over time.

Related: [[forgetting-curve]] [[anki-tips]]
```

### Map

Maps of Content (MOCs) - index notes that organize related content.

```yaml
---
title: Productivity MOC
type: map
tags: [productivity]
---

A curated collection of productivity-related content:

## Books
- [[atomic-habits]]
- [[deep-work]]

## Concepts
- [[habit-stacking]]
- [[time-blocking]]
```

## Adding New Content Types

To add a new content type:

1. Update `content.config.ts` - add to `externalContentTypes` or `personalContentTypes` array
2. Add any type-specific fields to the schema
3. Update type icons in `app/constants/contentTypes.ts`
4. Add UI handling in relevant pages/components

See [Extending the App](/customization/extending) for detailed instructions.
