# Content System

Second Brain uses a flat content structure powered by [@nuxt/content v3](https://content.nuxt.com/). All content lives as Markdown files with YAML frontmatter.

## Philosophy

**Everything is flat.** Unlike traditional folder hierarchies, Second Brain follows the Zettelkasten methodology where structure emerges from connections, not folders.

- No nested directories for content
- Organization through wiki-links (`[[slug]]`)
- Tags for cross-cutting categorization
- Authors link content to creators

## How Content Works

Each piece of content is a Markdown file with two parts:

1. **Frontmatter** - YAML metadata at the top (title, type, tags, etc.)
2. **Body** - Markdown content with optional wiki-links

```md
---
title: Atomic Habits
type: book
authors: [james-clear]
tags: [productivity, habits]
rating: 7
readingStatus: finished
---

Great book on building habits through small changes.

Key connections: [[habit-stacking]] [[1-percent-better]]
```

## Collections

Content is organized into **collections** defined in `content.config.ts`:

| Collection | Purpose | Location |
|------------|---------|----------|
| `content` | Main knowledge base items | `content/*.md` |
| `authors` | Creator profiles | `content/authors/*.md` |
| `podcasts` | Podcast show definitions | `content/podcasts/*.md` |
| `newsletters` | Newsletter definitions | `content/newsletters/*.md` |
| `tweets` | Saved tweets | `content/tweets/*.md` |
| `pages` | Static pages (About, etc.) | `content/pages/*.md` |

## Content Types

Content items have a `type` field that determines their behavior and required fields:

**External Types** (require authors):
- `youtube`, `podcast`, `article`, `book`, `manga`, `movie`, `tv`, `tweet`, `course`, `reddit`, `github`, `newsletter`

**Personal Types** (authors optional):
- `quote`, `note`, `evergreen`, `map`

See [Content Types](/content/content-types) for detailed documentation of each type.

## Querying Content

In Vue components, query content using the `queryCollection` composable:

```ts
// Get all content
const { data } = await useAsyncData(() =>
  queryCollection('content').all()
)

// Filter by type
const { data } = await useAsyncData(() =>
  queryCollection('content')
    .where('type', '==', 'book')
    .all()
)
```
